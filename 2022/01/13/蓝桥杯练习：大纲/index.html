蓝桥杯练习：大纲

发表于 2022-01-13 分类于 蓝桥杯练习 阅读次数：

本章介绍了在比赛时如何判断应用哪一种算法以及进行算法判断的简便方法。

解题步骤第一步：抽象出模型

常见的算法模型有dfs，图论，dp，贪心等，看到题目第一反应就是把实际的应用题转化为对应的抽象模型。

这个方面没有什么捷径可言，只能多积累，多刷题，省赛想出成绩至少要刷200道题，国赛的话至少300道题。

解题步骤第二步：考虑时间复杂度

判断时间是否会超时

C++在比赛的编译器上，一秒运行一亿次，也就是说运算次数小于10^7~10^8，就是可以的，每道题都有一个数据范围，对应的，基本上看到数值范围就可以知道这道题可以运用什么算法来解决，对应的算法及时间复杂度如下

  1. n<=30，指数级别，dfs加剪枝，状态压缩dp
  2. n <= 100 -> O(n^2)，
  3. n <= 1000 -> O(n^2)，O(n^2logn)，dp，二分
  4. n <= 10000 -> O(n^3/2)，块状链表
  5. n <= 100000 -> O(nlogn)，各种sourt，线段树，树状数组，set/map，heap，dijsdtra+heap，spfa，求凸包，平面交，二分
  6. n <= 1000000 -> O(n)，以及常数较小的O(nlogn)算法，hash，双指针扫描，kmp，AC自动机，O（nlogn）的做法：sort，树状数组，heap，dijkstra，spfa
  7. n <= 10000000 -> O(n)，双指针扫描，kmp，AC自动机
  8. n <= 100000000 -> O(n^1/2)，判断质数
  9. n <= 10^18 -> O(logn)，最大公约数

记住一些变量类型的数值范围

int：-2147483648 ~ 2147483647

long long：-10^18 ~ 10^18
